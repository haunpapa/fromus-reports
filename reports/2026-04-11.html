항목링크

2 GitHub 리포지토리 만들기

2-1. 새 리포 생성

  • github.com 로그인 → 우측 상단 + → New repository
  • Repository name: fromus-reports (원하는 이름)
  • Public 선택 (GitHub Pages 무료는 Public만 됨)
  • "Add a README file" 체크
  • Create repository 클릭

2-2. GitHub Pages 활성화

  • 생성된 리포 → Settings 탭 클릭
  • 좌측 메뉴에서 Pages 클릭
  • Source: Deploy from a branch
  • Branch: main / / (root) 선택
  • Save 클릭
1~2분 후 https://[아이디].github.io/fromus-reports/ 주소가 활성화됩니다.

3 GitHub Desktop으로 클론

  • GitHub Desktop 실행 → File → Clone Repository
  • 방금 만든 fromus-reports 선택
  • Local Path 확인 (예: C:\Users\동이\Documents\GitHub\fromus-reports)
  • Clone 클릭
이제 이 로컬 폴더가 GitHub과 연결된 작업 폴더입니다.

4 폴더 구조 만들기

클론한 폴더 안에 이렇게 파일을 배치하세요:

fromus-reports/
├── index.html          ← 자동 생성됨
├── build_index.py     ← 인덱스 생성 스크립트
├── reports/
│   ├── 2026-04-06.html
│   ├── 2026-04-07.html
│   ├── 2026-04-08.html
│   ├── 2026-04-09.html
│   └── 2026-04-10.html
└── README.md
파일명 규칙: YYYY-MM-DD.html 형식으로 통일하세요.
예: 2026-04-10.html, 2026-04-11.html ...

5 첫 배포

5-1. 인덱스 생성

# 클론한 폴더에서 터미널(또는 명령 프롬프트) 열기
cd C:\Users\동이\Documents\GitHub\fromus-reports

# 인덱스 자동 생성
python build_index.py

# 결과:
━━━ 프롬어스 리포트 인덱스 생성기 ━━━
📂 스캔 폴더: reports/
✅ 5개 리포트 발견
🎉 index.html 생성 완료!

5-2. GitHub Desktop에서 Push

  • GitHub Desktop 열기 → 변경된 파일 목록 확인
  • 좌측 하단 Summary에 첫 리포트 업로드 입력
  • Commit to main 클릭
  • 상단 Push origin 클릭
  • 1~2분 후 사이트 확인!
완료! https://[아이디].github.io/fromus-reports/ 에서 확인하세요.

6 매일 하는 일 (2분)

이제부터 매일 반복하는 건 이것뿐이에요:

# 1. 새 리포트를 reports/ 폴더에 저장
# 예: reports/2026-04-11.html

# 2. 인덱스 갱신
python build_index.py

# 3. GitHub Desktop 열기 → Commit → Push
# 끝! 🎉

자주 묻는 질문

카카오톡에 공유할 URL은?

항상 같은 URL 하나만 공유하면 됩니다:
https://[아이디].github.io/fromus-reports/
멤버들이 여기서 최신 + 과거 리포트를 모두 볼 수 있어요.

특정 날짜 리포트 직접 링크는?

https://[아이디].github.io/fromus-reports/reports/2026-04-10.html

커스텀 도메인 연결 가능?

네! 리포 Settings → Pages → Custom domain에 도메인 입력하면 됩니다.
예: report.fromus.co.kr (도메인 DNS에서 CNAME 설정 필요)

비용은?

완전 무료예요. GitHub Pages는 Public 리포 기준 빌드·대역폭 모두 무제한(소프트 리밋 100GB/월)입니다.

비공개로 운영하고 싶으면?

GitHub Pro($4/월) 또는 GitHub Teams 플랜이면 Private 리포에서도 GitHub Pages 사용 가능합니다.
